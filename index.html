SECT 05

Как звать в работе

commands · slash

После установки плагинов у тебя в Claude Code появляются команды. Вот основные.

  • /codex:review — ревью кода. Claude пишет, Codex проверяет, выдаёт High/Medium/Low
  • /codex:adversarial-review — жёсткое ревью. Codex играет в дьявола-адвоката
  • /codex:rescue "задача" — спасение. Если Claude застрял, Codex решает с нуля
  • /codex:review --background — в фоновом режиме (не блокирует чат)
  • /cc-gemini-plugin:gemini "задача" @file.mp4 — Gemini анализ файла
  • /cc-gemini-plugin:gemini --dirs src,docs "найди X" — Gemini сканирует целые папки

PROMPT 01 · Email-рассыльщик (build)
Сделай Python-скрипт для рассылки писем по списку email из CSV файла. Subject и body — аргументы. SMTP-кредентилы из .env. Логирует отправленные.
PROMPT 02 · Мульти-ревью Codex (4 направления, реальный проект)
Прогони этот скрипт через codex review. Проверь четыре направления: 1. SECURITY: пароли в логах, токены, нет TLS, валидация ввода, инъекции 2. ЛОГИ: уровни, конфиг, ротация, что светится в stdout Дай разбор по каждому пункту в формате High / Medium / Low. В конце два-три самых критичных фикса которые надо сделать сразу.

SECT 10

Если что-то не работает

troubleshoot · 8 кейсов

Реальные грабли которые я сам собрал во время настройки. Особенно на Windows. Найди свою проблему — копируй фикс.

  • На Windows установка PATH не работает автоматически. Это норма — npm складывает CLI в C:\npm-global, но Windows не добавляет эту папку в PATH сам. Нужно один раз настроить вручную.
  • Любую env-переменную после установки нужно «закрепить» через системные переменные. Иначе она живёт только до закрытия терминала.
  • После установки PATH/env обязательно перезапусти Claude Code и терминал. Они стартуют со снапшотом env.
ERR 01
'codex' is not recognized / 'gemini' is not recognized
FIX · add C:\npm-global to current session PATH
$env:Path += ";C:\npm-global"
FIX · сделать PATH постоянным (опционально)
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\npm-global", "User")
Cause
папка C:\npm-global не в PATH. Первая команда добавляет её в текущую сессию терминала — работает сразу.
Permanent
Вторая команда прописывает PATH в системные User-переменные навсегда. Но текущая сессия про это не узнает — нужно вводить первую команду в каждом новом терминале, либо просто перезапустить Antigravity целиком.
Tested
Если перезапуск Antigravity не помог (бывает — встроенный терминал кэширует) — используй первую команду. Гарантированно работает.
ERR 02
«Session ended · invalid_state» при codex login
FIX · wipe codex cache + retry
Remove-Item -Recurse -Force "$env:USERPROFILE\.codex" -ErrorAction SilentlyContinue
codex login
Cause
OAuth-сессия в браузере истекла пока CLI ждал callback. Снос кэша Codex решает.
Alt
codex login --device-auth — если терминал не открывает браузер автоматически.
ERR 03
SMS-код при регистрации ChatGPT не приходит
FIX · go API-key route instead
$env:OPENAI_API_KEY = "sk-..."
[Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "sk-...", "User")
Cause
SMS-доставка в РФ/СНГ часто кривит. Иди через API-ключ.
Steps
platform.openai.com/signup через Google (без SMS) → пополни баланс на $5 → создай API-ключ → подставь в команду выше.
ERR 04
401 Unauthorized «Missing bearer» даже когда ключ установлен
FIX · wipe broken cached credential, re-login as API key
Remove-Item -Recurse -Force "$env:USERPROFILE\.codex" -ErrorAction SilentlyContinue
codex login
# выбери "Use an API key" → вставь свой sk-...
Cause
Codex имеет битый кэш credential от прошлой OAuth-попытки и игнорирует OPENAI_API_KEY из env (известный баг — GitHub issue #3286).
Fix
снести кэш и залогиниться заново через интерактивный режим — выбрать «Use an API key» стрелочками и вставить ключ.
ERR 05
'/plugin' is not recognized в терминале
FIX · launch Claude Code first
claude
Cause
команды /plugin ... — это команды Claude Code, не PowerShell/zsh. Нужно сначала запустить Claude Code командой claude прямо в терминале.
Then
внутри Claude Code вводи /plugin marketplace add openai/codex-plugin-cc и далее.
ERR 06
«Please run /login · 401 Invalid authentication credentials» в Claude Code
FIX · re-auth Claude Code session
/login
Cause
сессия Claude Code (Anthropic) истекла. Плагины не доходят до своих ключей пока Claude Code сам не залогинен.
Diag
если не помогло — /doctor для полной диагностики.
ERR 07
/cc-gemini-plugin:gemini молча возвращает пустой результат на Windows
FIX · ask Claude to patch the bridge script
почини баг в bridge-скрипте
Cause
реальный баг в плагине на line 448 — import.meta.url на Windows возвращает file:///C:/..., а process.argv[1] — file://C:\.... Несовпадение форматов — main() никогда не запускается.
Fix
попроси Claude этой фразой — он сам поправит файл C:\Users\...\.claude\plugins\cache\cc-gemini-plugin\...\scripts\gemini-bridge.js. Три правки: pathToFileURL, промпт через stdin, cmd.exe /d /s /c gemini.
Note
правка живёт в кэше плагина — обновление плагина её затрёт. После обновления попроси Claude починить заново.
ERR 08
Ничего из вышеперечисленного не помогло
FIX · run full diagnostic
/doctor
Action
Claude Code запустит полную диагностику окружения — версии CLI, кэшированные credentials, состояние плагинов, PATH, скиллы.
Help
скинь скриншот вывода в мой бот по первой ссылке в описании — разберёмся.
